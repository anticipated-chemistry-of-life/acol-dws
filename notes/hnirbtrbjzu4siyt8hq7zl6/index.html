Discretization of branch lengths

The parameters of the rate matrix μc1\mu_{c1}μc1​ and μc2\mu_{c2}μc2​ and the branch lengths b(n)b(n)b(n) are non identifiable: doubling all rate parameters and halving all branch lengths will lead to the exact same solution. We therefore introduce the constraint:

∑nb(n)=1\sum_n{b(n)} = 1n∑​b(n)=1

such that 0≤b(n)≤10 \leq b(n) \leq 10≤b(n)≤1 for all branch lengths b(n)b(n)b(n). We further note that calculating the matrix exponential P(n)=exp⁡(Λcb(n))\mathbf{P}(n) = \exp(\mathbf{\Lambda_c}b(n))P(n)=exp(Λc​b(n)) for every possible branch length is computationally prohibitive. To reduce the number of calculations, we bin the branch lengths to predefined values. Specifically, let there be a regularly spaced grid on the interval [a,b][a, b][a,b] consisting of KKK bins. Discretization of rate parameters

When updating the transition rate parameters μc1\mu_{c1}μc1​ and μc2\mu_{c2}μc2​ for a clique ccc, the transition matrix P(k)\mathbf{P}(k)P(k) for all k=0,...,K−1k = 0,..., K-1k=0,...,K−1 bins must be re-calculated. Despite the above approach, this becomes computationally prohibitive when considering there to be millions of cliques (one per molecule). We therefore propose to discretize the values of the rate parameters as well. We will use a regularlly spaced grid in the interval [x,y][x,y][x,y] with a total of MMM bins. It is still not clear how many bins we will need. It is then possible to pre-calculate and store all combinations of K branch lengths and M2M^2M2 values of μc1\mu_{c1}μc1​ and μc2\mu_{c2}μc2​, such that the update of these parameters will be very fast.
